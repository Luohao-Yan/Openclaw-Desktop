## 一、权限体系设计
### 1.1 角色权限划分
| 角色 | 权限范围 |
|------|----------|
| 所有者（Owner） | 全部权限：Workspace创建/删除/配置、成员管理、文档编辑/删除、版本回滚、权限调整 |
| 管理员（Admin） | 成员管理（仅可添加/移除普通成员）、文档编辑/删除、版本查看、模板配置 |
| 普通成员（Member） | 文档编辑、版本查看、自己创建内容的删除权限 |
| 访客（Guest） | 只读权限，仅可查看已开放内容，不可编辑/删除 |

### 1.2 资源权限粒度
- Workspace级：控制整个工作空间的访问权限
- 目录级：控制单个目录下所有内容的访问权限
- 文件级：控制单个文档/文件的访问权限
- 片段级：支持对文档中指定片段设置单独的编辑/查看权限

### 1.3 权限继承规则
- 下级资源默认继承上级资源权限配置
- 单独配置的权限优先级高于继承权限
- 权限仅可向下收紧，不可向上突破（例：父目录设置为只读，子目录不可设置为可编辑）

## 二、需求边界规范
### 2.1 版本回溯规范
- 所有文档自动保存历史版本，版本保留周期默认90天（可配置最长365天）
- 版本记录包含：修改人、修改时间、修改内容Diff、版本备注
- 支持一键回滚到任意历史版本，回滚操作会生成新的版本记录，不会覆盖原有版本链
- 仅所有者和管理员可执行版本回滚操作，普通成员仅可查看版本历史

### 2.2 编辑冲突处理规范
- 实时协作编辑采用OT算法，默认自动合并无冲突的修改
- 出现无法自动合并的冲突时，弹窗提示用户选择：保留我的修改/保留他人修改/手动合并
- 冲突修改会自动保存为临时版本，避免内容丢失
- 冲突记录会保留在版本历史中，可追溯冲突处理过程

### 2.3 默认模板规范
- 系统内置3类默认模板：需求文档模板、技术方案模板、会议纪要模板
- 支持Workspace管理员自定义模板，自定义模板仅在当前Workspace内可用
- 新建文档时默认展示推荐模板列表，用户可选择模板或空白文档
- 模板支持设置默认权限配置，新建文档自动继承模板的权限设置
