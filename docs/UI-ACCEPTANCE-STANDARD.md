## 一、UI验收检查清单（12项）

### 1. 视觉一致性检查

| # | 检查项 | 验收标准 | 优先级 |
|---|-------|---------|--------|
| 1 | **配色方案** | 主色 `#3B82F6`，背景 `#0F172A`，侧边栏 `#1E293B` 正确应用 | P0 |
| 2 | **字体统一** | 使用系统字体栈 `-apple-system, BlinkMacSystemFont...`，无乱码 | P0 |
| 3 | **间距一致性** | 卡片内边距 `p-6`，标题 `text-2xl`，间隙 `gap-6` 统一 | P1 |
| 4 | **圆角规范** | 卡片 `rounded-lg`，按钮 `rounded` 或 `rounded-full` 一致 | P1 |

### 2. 深色主题专项检查（重点）

| # | 检查项 | 验收标准 | 优先级 |
|---|-------|---------|--------|
| 5 | **主文字可读性** | 深色背景 `#F1F5F9` 文字与 `#0F172A` 背景对比度 ≥ 15:1 | P0 |
| 6 | **次要文字可读性** | `#94A3B8` 次要文字清晰可读，无模糊 | P0 |
| 7 | **状态颜色对比** | 成功 `#22C55E` / 警告 `#F59E0B` / 错误 `#EF4444` 在深色背景明显可辨 | P0 |
| 8 | **输入框对比** | 输入框背景 `#1E293B` 与文字 `#F1F5F9` 对比度 ≥ 12:1 | P0 |

### 3. 交互状态检查

| # | 检查项 | 验收标准 | 优先级 |
|---|-------|---------|--------|
| 9 | **Hover状态** | 按钮 hover 时背景色变化明显，cursor 变为 pointer | P1 |
| 10 | **Active/Focus状态** | 输入框 focus 时有明显的边框或阴影反馈 | P1 |
| 11 | **Disabled状态** | 禁用按钮 `opacity-50` 且 `cursor-not-allowed`，不可点击 | P1 |
| 12 | **Loading状态** | 操作按钮 loading 时有 spinner 或禁用状态，防止重复提交 | P1 |

### 4. 响应式与布局检查

| # | 检查项 | 验收标准 | 优先级 |
|---|-------|---------|--------|
| 13 | **窗口缩放** | 最小窗口 800x600 时所有内容可见，无重叠或截断 | P1 |
| 14 | **侧边栏折叠** | 侧边栏宽度 200px，可折叠，折叠后图标可见 | P2 |
| 15 | **滚动条样式** | 自定义滚动条样式统一，hover 时 thumb 颜色变化 | P2 |

## 二、快速验收流程（30分钟）

### Step 1: 视觉检查（10分钟）
```bash
# 启动应用
npm run dev
```
- [ ] 打开 Dashboard、Config、Tasks、Logs、Settings 五个页面
- [ ] 检查深色主题是否正确渲染（背景应为 `#0F172A`）
- [ ] 检查所有文字是否清晰可读，无模糊或锯齿

### Step 2: 深色主题专项测试（10分钟）
- [ ] 在较暗环境（夜间模式）下检查所有页面
- [ ] 检查 Gateway 状态徽章（绿色/红色/黄色）在深色背景是否明显
- [ ] 检查错误提示框 `bg-red-900/30` 中的文字可读性
- [ ] 检查输入框中 placeholder 文字颜色是否过淡

### Step 3: 交互测试（5分钟）
- [ ] 测试所有按钮的 hover 效果
- [ ] 测试输入框的 focus 状态
- [ ] 测试禁用按钮（如 loading 状态下）不可点击
- [ ] 测试侧边栏导航切换

### Step 4: 响应式测试（5分钟）
- [ ] 将窗口缩放到 800x600 检查布局
- [ ] 检查长列表（Tasks/Logs）滚动是否正常
- [ ] 检查表格内容是否截断

## 三、常见问题提醒

### 🚨 高风险点（易遗漏）

1. **Placeholder 对比度不足**
   - 问题：深色主题下 placeholder 文字颜色过淡，几乎看不见
   - 检查位置：Config 页面的所有输入框
   - 验收：`placeholder-slate-400` 或更高对比度

2. **错误信息可读性**
   - 问题：`bg-red-900/30` + `text-red-400` 在深色背景对比度不够
   - 检查位置：Dashboard 的错误提示框
   - 验收：确保文字清晰可读

3. **状态颜色在深色背景下的表现**
   - 问题：绿色/黄色/红色在深色背景上可能不够鲜明
   - 检查位置：Gateway 状态徽章、任务状态标签
   - 验收：增加边框或提高饱和度

4. **滚动条与内容重叠**
   - 问题：自定义滚动条可能遮挡内容
   - 检查位置：Logs 页面长日志列表
   - 验收：预留滚动条空间，无内容遮挡

5. **按钮 Loading 状态反馈**
   - 问题：点击后无反馈，用户可能重复点击
   - 检查位置：Dashboard 的 Start/Stop/Restart 按钮
   - 验收：loading 时有 spinner 或禁用状态

### ⚠️ 中风险点

6. **Hover 状态不明显**
   - 问题：`hover:bg-slate-700` 与 `bg-slate-800` 差异太小
   - 验收：hover 时颜色变化 ≥ 100 色阶

7. **Focus 环缺失**
   - 问题：键盘导航时不知道当前焦点在哪
   - 验收：所有可交互元素有可见 focus ring

8. **窗口大小变化时的闪烁**
   - 问题：快速调整窗口大小时布局抖动
   - 验收：流畅过渡，无闪烁

## 六、工具推荐

如需精确检查对比度：
1. **浏览器 DevTools** → Elements → 选中元素 → 查看颜色对比度提示

3. **macOS 自带**: 系统设置 → 辅助功能 → 显示 → 增加对比度（模拟测试）
