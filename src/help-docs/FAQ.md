## 一、安装启动类问题
### Q1：安装包下载后无法打开，提示"无法验证开发者"（macOS）
**问题现象**：打开dmg安装包时弹出提示"无法打开'OpenClaw Desktop'，因为无法验证开发者"
**解决方案**：
1. 右键点击安装包，选择"打开"
2. 在弹出的提示框中再次点击"打开"即可正常安装
3. 若仍无法打开，进入「系统设置」→「隐私与安全性」，在"安全性"部分点击"仍要打开"按钮

### Q2：Windows安装时提示"Windows已保护你的电脑"
**问题现象**：运行exe安装包时弹出SmartScreen拦截提示
**解决方案**：
1. 点击提示框中的"更多信息"
2. 点击"仍要运行"按钮即可继续安装
3. 若需要永久放行，可在安装文件右键→「属性」→勾选"解除锁定"后再运行

### Q3：首次启动提示"未检测到OpenClaw核心服务"
**问题现象**：启动应用后顶部状态显示红色，提示核心服务未运行
**解决方案**：
1. 点击仪表板上的"一键修复"按钮，系统会自动下载并配置核心服务
2. 若修复失败，检查是否开启了VPN/代理，临时关闭后重试
3. 手动安装核心服务：打开终端执行 `npm install -g openclaw` 后重启应用

### Q4：Linux下AppImage文件无法运行
**问题现象**：双击AppImage文件无反应
**解决方案**：
1. 右键点击文件→「属性」→「权限」→勾选"允许作为程序执行"
2. 或在终端执行：`chmod +x OpenClaw-Desktop-x.x.x.AppImage`
3. 再双击运行即可

### Q5：启动后界面空白，一直加载中
**问题现象**：启动后显示白屏或加载动画一直转圈
**解决方案**：
1. 关闭应用后重新启动，90%的情况可以解决
2. 若仍空白，删除缓存目录：
   - macOS：`rm -rf ~/Library/Application Support/openclaw-desktop/Cache`
   - Windows：`rmdir /s /q %APPDATA%\openclaw-desktop\Cache`
   - Linux：`rm -rf ~/.config/openclaw-desktop/Cache`
3. 检查显卡驱动是否为最新版本，关闭硬件加速：在设置→高级设置中开启"禁用硬件加速"

## 二、运行故障类问题
### Q6：系统运行时经常卡顿，CPU占用过高
**问题现象**：使用过程中应用卡顿，任务管理器显示CPU占用超过50%
**解决方案**：
1. 检查是否同时运行了多个智能体会话，关闭不需要的会话
2. 在设置→通用设置中开启"低功耗模式"，降低后台资源占用
3. 检查是否有大量定时任务在同时执行，暂停不需要的任务
4. 若使用的是大尺寸模型，切换到较小的模型可以大幅降低资源占用

### Q7：服务异常退出，提示"核心服务崩溃"
**问题现象**：使用过程中突然弹出提示"核心服务异常退出"
**解决方案**：
1. 点击"重启服务"按钮，等待服务重启完成即可恢复
2. 查看日志管理中的错误日志，确认崩溃原因
3. 若频繁崩溃，在设置→高级设置中开启"调试模式"，提交日志给技术支持排查
4. 检查是否有杀毒软件/安全软件拦截了OpenClaw的进程，将应用加入白名单

### Q8：一键修复功能失效，无法自动修复问题
**问题现象**：点击"一键修复"后提示修复失败
**解决方案**：
1. 检查网络连接是否正常，能否访问GitHub和npm仓库
2. 手动执行修复命令：打开终端执行 `openclaw doctor`，根据提示修复问题
3. 若仍无法解决，卸载应用后重新安装最新版本
4. 注意：重装应用不会丢失已有的智能体和会话数据，数据保存在`~/.openclaw/`目录下

### Q9：应用更新失败，无法升级到最新版本
**问题现象**：收到更新提示后点击更新，提示更新失败
**解决方案**：
1. 检查网络连接，关闭代理后重试
2. 手动下载最新安装包覆盖安装即可，无需卸载旧版本
3. macOS用户若提示"权限不足"，将应用从应用程序文件夹拖到桌面再执行更新
4. 若仍失败，备份配置后删除`~/.openclaw/desktop/config.json`文件后重启应用

## 三、智能体相关问题
### Q10：创建智能体时没有我需要的模型选项
**问题现象**：模型下拉列表中找不到想要使用的大模型
**解决方案**：
1. 进入「设置」→「模型配置」，点击"添加模型"
2. 选择对应的模型厂商，填写API Key和相关配置
3. 保存后回到智能体创建页面，即可看到新添加的模型
4. 注意：部分模型需要申请API访问权限，请确保已经在对应厂商平台开通了服务

### Q11：智能体回答不准确，总是答非所问
**问题现象**：智能体的回答不符合预期，经常出现幻觉或错误信息
**解决方案**：
1. 在智能体编辑页面调整模型参数：降低"温度系数"（建议设置为0.1-0.3），减少随机性
2. 完善智能体的System Prompt，明确告知智能体的角色和回答要求
3. 绑定知识库插件，让智能体基于指定的知识库内容回答
4. 若使用的是较小的模型，切换到更大参数的模型可以显著提升回答质量

### Q12：智能体导入失败，提示"配置文件格式错误"
**问题现象**：导入外部智能体配置文件时提示格式错误
**解决方案**：
1. 检查配置文件是否为有效的JSON格式，有没有语法错误
2. 确认配置文件是OpenClaw的智能体导出格式，不要导入其他平台的配置文件
3. 若文件损坏，尝试重新获取原始导出文件
4. 可以手动创建智能体，再对照配置文件的参数手动填写

### Q13：删除智能体后数据还能恢复吗？
**问题现象**：不小心删除了重要的智能体，想要恢复
**解决方案**：
1. 智能体删除后会进入回收站，7天内可以在「智能体管理」→「回收站」中恢复
2. 若超过7天，且之前做过导出备份，可以导入备份文件恢复
3. 若没有备份，数据无法恢复，请谨慎操作删除功能
4. 建议定期导出重要智能体的配置文件做备份

### Q14：智能体无法调用绑定的技能
**问题现象**：已经给智能体绑定了技能，但智能体不会主动调用
**解决方案**：
1. 检查技能是否已经正确配置，在技能详情页点击"诊断"确认技能运行正常
2. 在智能体的System Prompt中明确告知智能体可以使用哪些技能，以及什么时候使用
3. 检查智能体的工具调用权限是否开启，在智能体编辑页面→高级设置中确认"允许调用工具"已开启
4. 部分模型对工具调用的支持不好，切换到支持函数调用的模型（如GPT-4、Claude 3、Doubao Pro等）

## 四、任务管理相关问题
### Q15：定时任务到点不执行
**问题现象**：设置了定时任务，但到了设定时间没有执行
**解决方案**：
1. 检查任务是否处于"启用"状态，开关是否打开
2. 检查CRON表达式是否正确，可以使用在线CRON表达式验证工具确认
3. 检查关联的智能体是否存在且运行正常
4. 查看任务的执行历史，查看错误信息定位问题
5. 注意：若应用处于关闭状态，定时任务不会执行，需要保持应用在后台运行

### Q16：任务执行失败，提示"会话不存在"
**问题现象**：任务执行日志中提示"会话不存在"错误
**解决方案**：
1. 任务关联的会话可能已经被删除，编辑任务重新选择关联的会话
2. 若不需要关联特定会话，创建任务时选择"创建新会话"选项
3. 检查智能体是否正常运行，重新绑定智能体后重试

### Q17：最多可以创建多少个定时任务？
**问题现象**：创建任务时提示已达上限
**解决方案**：
1. 免费版本最多支持创建20个定时任务，专业版本无限制
2. 删除不需要的旧任务，释放配额
3. 若需要更多任务，可以联系销售升级到专业版本
4. 对于简单的重复任务，可以合并为一个任务减少占用

### Q18：任务执行结果如何通知我？
**问题现象**：想知道任务执行成功还是失败，但找不到通知入口
**解决方案**：
1. 在任务编辑页面可以配置通知方式：
   - 应用内通知：任务执行完成后会在系统通知栏弹出提示
   - 消息通道通知：可以配置发送执行结果到飞书、Telegram等通道
   - 邮件通知：填写邮箱地址，执行结果会发送到指定邮箱
2. 可以随时在任务详情页查看所有历史执行记录和结果

## 五、技能管理相关问题
### Q19：技能安装失败，提示"网络错误"
**问题现象**：从市场安装技能时提示网络错误，安装失败
**解决方案**：
1. 检查网络连接是否正常，能否访问ClawHub市场
2. 若开启了代理，关闭代理后重试，或配置代理的白名单
3. 手动下载技能安装包，在技能管理页面点击"导入技能"手动安装
4. 检查磁盘空间是否足够，至少预留100MB以上的可用空间

### Q20：技能安装后无法使用，提示"缺少配置参数"
**问题现象**：技能已经安装成功，但使用时提示缺少配置
**解决方案**：
1. 进入技能详情页，查看"配置要求"部分，填写所有必填的配置项（如API Key、访问地址等）
2. 配置完成后点击"保存"，再点击"诊断"按钮确认配置正确
3. 部分技能需要付费使用，确认已经购买了对应的服务权限
4. 查看技能文档，确认使用方法是否正确

### Q21：如何开发自己的自定义技能？
**问题现象**：市场上没有我需要的技能，想自己开发
**解决方案**：
1. 参考官方开发文档：https://docs.openclaw.ai/skill-development/
2. 点击技能管理页面的"新建技能"，选择模板后即可开始开发
3. 技能支持JavaScript/TypeScript开发，门槛低，有编程基础即可快速上手
4. 开发完成后可以发布到ClawHub市场分享给其他用户

### Q22：技能更新会丢失原有配置吗？
**问题现象**：更新已安装的技能，之前的配置会不会被清空
**解决方案**：
1. 正常升级技能不会丢失原有配置，配置参数会自动保留
2. 若技能的新版本有破坏性变更，会提示你确认是否要迁移配置
3. 建议更新前导出技能配置做备份，防止意外
4. 若更新后技能无法使用，可以卸载后重新安装配置

## 六、会话管理相关问题
### Q23：会话记录会永久保存吗？占空间太大怎么办？
**问题现象**：会话越来越多，担心占用太多磁盘空间
**解决方案**：
1. 会话记录默认永久保存，除非你手动删除
2. 可以在设置→通用设置中配置"会话自动清理"：设置保留时间，超过时间的会话会自动删除
3. 可以手动删除不需要的会话，释放空间
4. 重要会话可以导出为Markdown/JSON文件备份后再删除

### Q24：会话中的文件/附件保存在哪里？
**问题现象**：会话中上传的文件，本地路径在哪里
**解决方案**：
1. 会话附件默认保存在 `~/.openclaw/desktop/sessions/<会话ID>/attachments/` 目录下
2. 可以在设置→通用设置中修改默认的附件保存路径
3. 删除会话时，对应的附件也会被同步删除
4. 若需要长期保存附件，可以导出会话时选择"包含附件"选项

### Q25：可以同时开启多少个会话？
**问题现象**：开启多个会话会不会影响性能
**解决方案**：
1. 理论上没有数量限制，但建议同时开启的会话不要超过10个，避免占用过多资源
2. 每个会话是独立的，互相不会影响
3. 长时间不用的会话建议关闭，释放内存
4. 专业版本支持更多并发会话，适合重度用户

### Q26：会话导出的文件可以再导入回来吗？
**问题现象**：导出的会话备份文件，能不能恢复到应用中
**解决方案**：
1. 目前仅支持导入智能体配置文件，会话备份文件暂时不支持导入
2. 导出的Markdown文件可以直接用编辑器打开查看内容
3. 后续版本会支持会话导入功能，请关注更新日志
4. 若需要迁移会话，可以手动复制`~/.openclaw/desktop/sessions/`目录到新设备

## 七、系统设置相关问题
### Q27：修改配置后需要重启应用吗？
**问题现象**：修改了设置项，不知道要不要重启生效
**解决方案**：
1. 大部分配置（如主题、语言、通知设置）修改后实时生效，无需重启
2. 核心配置（如模型配置、通道配置、端口设置）修改后需要重启服务生效
3. 修改配置后页面会提示是否需要重启，按照提示操作即可
4. 若不确定是否生效，重启应用是最稳妥的方式

### Q28：飞书/电报通道配置失败，消息发不出去
**问题现象**：配置了消息通道，但测试消息发送失败
**解决方案**：
1. 检查参数是否填写正确：
   - 飞书：确认App ID、App Secret、加密密钥等参数正确，机器人已经加入到对应群组
   - Telegram：确认Bot Token正确，机器人已经和用户发起过对话
2. 检查网络是否能访问对应平台的接口，必要时配置代理
3. 点击"测试发送"按钮，根据返回的错误信息排查问题
4. 参考官方文档的配置教程：https://docs.openclaw.ai/channels/

### Q29：数据存在本地安全吗？会不会上传到云端？
**问题现象**：担心智能体和会话数据泄露，会不会被上传
**解决方案**：
1. OpenClaw是本地优先的架构，所有数据默认保存在本地，不会主动上传到任何云端服务器
2. 只有当你主动开启云同步功能时，数据才会加密后同步到你自己的云存储
3. 智能体调用大模型时，只有当前会话的上下文会发送给模型厂商，不会发送其他数据
4. 你可以完全控制自己的数据，不需要担心隐私泄露问题

### Q30：如何备份所有配置和数据？
**问题现象**：要换电脑，如何迁移所有数据到新设备
**解决方案**：
1. 最简单的方式：直接拷贝整个`~/.openclaw/`目录到新设备的相同路径
2. 选择性备份：
   - 智能体配置：`~/.openclaw/agents/`
   - 会话记录：`~/.openclaw/desktop/sessions/`
   - 技能配置：`~/.openclaw/skills/`
   - 系统配置：`~/.openclaw/config/`
3. 新设备安装应用后，将备份的目录覆盖对应路径即可恢复所有数据
4. 专业版本支持云同步功能，可以自动在多设备间同步数据

## 八、网络与权限类问题
### Q31：公司内网环境下无法使用，提示网络错误
**问题现象**：在公司内网环境下，核心服务无法启动，技能市场无法访问
**解决方案**：
1. 联系公司IT部门，将以下域名加入白名单：
   - `api.openclaw.ai` 官方API
   - `clawhub.ai` 技能市场
   - `registry.npmjs.org` npm包仓库
   - 你使用的大模型厂商的API域名（如`openai.com`、`doubao.volcengine.com`等）
2. 在设置→网络设置中配置公司代理，支持HTTP/HTTPS/SOCKS5代理
3. 若仍无法使用，可以申请使用离线版本，不需要连接外网

### Q32：调用大模型提示"API请求失败"
**问题现象**：智能体回复时提示API请求失败，无法获取回答
**解决方案**：
1. 检查API Key是否填写正确，有没有复制多余的空格
2. 检查API余额是否充足，在对应厂商的控制台查看余额
3. 检查网络是否能访问模型厂商的API接口，必要时配置代理
4. 检查模型是否支持你所在的地区访问，部分模型需要特定地区的IP才能访问

### Q33：防火墙/杀毒软件拦截了应用怎么办？
**问题现象**：安全软件提示OpenClaw是危险程序，拦截了功能
**解决方案**：
1. 将OpenClaw Desktop应用和`~/.openclaw/`目录加入杀毒软件的白名单
2. 在防火墙规则中允许OpenClaw访问网络
3. 从官方渠道下载安装包，避免下载到被篡改的版本
4. 若确定是误报，可以提交安装包给杀毒软件厂商审核解除拦截

### Q34：代理配置后不生效怎么办？
**问题现象**：配置了代理，但还是无法访问外网资源
**解决方案**：
1. 确认代理地址和端口填写正确，测试代理是否能正常访问网络
2. 若使用的是SOCKS5代理，确认代理支持UDP转发
3. 重启应用后重试，部分代理配置需要重启生效
4. 可以尝试设置系统全局代理，应用会自动使用系统代理配置
