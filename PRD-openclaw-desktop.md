## 1. 产品背景和目标

### 1.1 背景

将原有的 Web 管理界面重构为 **Electron Mac 桌面应用**，实现真正的本地化管理体验。用户无需运行独立的后端服务，直接通过桌面应用可视化管理 OpenClaw。

### 1.2 核心目标

| 目标 | 描述 | 成功指标 |
|------|------|----------|
| **零配置启动** | 打开应用即可使用，无需启动后端服务 | 首次启动 < 3 秒 |
| **可视化配置** | 替代手动编辑 YAML 配置文件 | 配置错误率降低 80% |
| **实时监控** | 本地任务、会话、日志实时查看 | 延迟 < 1 秒 |
| **原生体验** | 符合 macOS 设计规范的桌面应用 | 用户满意度 > 4.5/5 |

## 2. 技术架构（重大变更）

### 2.1 架构演进

```
Before (Web):
┌──────────┐     HTTP      ┌──────────┐     gRPC      ┌──────────┐
│  Web UI  │ ◄───────────► │  Backend │ ◄───────────► │ Gateway  │
└──────────┘               └──────────┘               └──────────┘

After (Electron Desktop):
┌─────────────────────────────────────────────────────────────────┐
│                    OpenClaw Desktop (Electron)                   │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐        │
│  │ Dashboard│  │ Config   │  │ Sessions │  │ Logs     │        │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘  └────┬─────┘        │
│       │             │             │             │               │
│       └─────────────┴──────┬──────┴─────────────┘               │
│                            │                                    │
│                    ┌───────▼────────┐                          │
│                    │  Node.js Main  │                          │
│                    │  (Electron)    │                          │
│                    └───────┬────────┘                          │
│                            │ child_process / file I/O          │
└────────────────────────────┼────────────────────────────────────┘
                             │
                    ┌────────▼─────────┐
                    │  OpenClaw CLI    │
                    │  (本地安装)       │
                    └────────┬─────────┘
                             │
        ┌────────────────────┼────────────────────┐
        │                    │                    │
   ┌────▼────┐         ┌─────▼─────┐       ┌─────▼────┐
   │ Sessions│         │  Skills   │       │  Agents  │
   └─────────┘         └───────────┘       └──────────┘
```

### 2.2 核心技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **Frontend** | React + TypeScript + Tailwind CSS | 渲染层 |
| **Desktop Shell** | Electron 30+ | 桌面容器 |
| **Main Process** | Node.js | 系统调用、文件 I/O |
| **本地通信** | child_process, fs, path | 直接调用本地 OpenClaw |
| **状态管理** | Zustand | 应用状态 |
| **构建** | electron-builder | 打包 dmg/pkg |

### 2.3 与 OpenClaw 集成方式

| 功能 | 集成方式 | 说明 |
|------|----------|------|
| 读取配置 | `fs.readFile()` | 直接读取 `~/.openclaw/` 下的 YAML |
| 写入配置 | `fs.writeFile()` + 校验 | 修改后保存，自动备份 |
| 执行命令 | `child_process.spawn()` | 调用 `openclaw` CLI |
| 实时日志 | `spawn('tail', ['-f', logfile])` | 流式读取日志文件 |
| 监控状态 | 定时轮询 + 文件监听 | 监控 `~/.openclaw/sessions/` |

## 4. 数据需求（本地文件系统）

### 4.1 数据来源

| 数据类型 | 来源路径 | 读取方式 |
|----------|----------|----------|
| 配置文件 | `~/.openclaw/*.yaml` | fs.readFile |
| 会话数据 | `~/.openclaw/sessions/` | fs.readdir + JSON parse |
| 日志文件 | `~/.openclaw/logs/` | fs.createReadStream |
| 技能列表 | `~/.openclaw/skills/` | fs.readdir |
| Agent 配置 | `~/.openclaw/agents/` | fs.readdir + YAML parse |

### 4.2 数据缓存策略

| 数据 | 缓存位置 | 更新策略 |
|------|----------|----------|
| 配置内容 | Renderer 内存 | 文件监听 + 手动刷新 |
| 会话列表 | Zustand Store | 轮询 2s + 文件监听 |
| 日志内容 | 滚动缓冲区 | 流式追加 |

### 4.3 配置备份

- **自动备份**: 每次保存配置前，自动创建 `.yaml.bak` 备份
- **备份保留**: 最近 10 个版本
- **一键回滚**: 支持从历史备份恢复

## 6. 用户体验设计要点

### 6.1 设计原则

1. **本地优先**: 所有操作即时生效，无需等待服务器响应
2. **容错设计**: 配置错误时友好提示，自动备份防止丢失
3. **实时反馈**: Gateway 状态、会话变化实时反映在 UI
4. **Mac 原生**: 遵循 macOS HIG，支持菜单栏、快捷键、Touch Bar

### 6.2 关键交互

| 场景 | 设计要点 |
|------|----------|
| 首次启动 | 检测 OpenClaw 安装状态，引导安装 |
| 配置编辑 | 表单化 + YAML 编辑器双模式 |
| 日志查看 | 支持搜索、过滤、导出 |
| 会话监控 | 卡片式展示，支持快速操作 |

## 8. 与原有 Web 方案的差异

| 对比项 | Web 方案 | Electron 方案 |
|--------|----------|---------------|
| 后端服务 | 需要独立 Node.js 后端 | ❌ 不需要 |
| 部署方式 | 浏览器访问 localhost | ✅ 独立桌面应用 |
| 本地集成 | 通过 API 间接访问 | ✅ 直接文件系统访问 |
| 用户体验 | 需要同时运行多个服务 | ✅ 一键启动 |
| 离线能力 | 依赖 Gateway | ✅ 完全离线可用 |
| 开发复杂度 | 高（前后端分离） | 中（Electron 单代码库） |

## 9. 附录

### 9.1 Electron 主进程职责

```typescript
// main.ts 核心职责
1. 创建 BrowserWindow
2. 注册 IPC handlers:
   - config:read, config:write
   - gateway:start, gateway:stop, gateway:status
   - sessions:list, sessions:get
   - logs:tail, logs:read
3. 文件系统监听（配置变更通知渲染进程）
4. 系统托盘 / 菜单栏集成
```

### 9.2 与 OpenClaw CLI 的调用约定

```bash
# 示例调用
openclaw gateway status --json          # 获取 Gateway 状态
openclaw sessions list --json           # 获取会话列表
openclaw agents list --json             # 获取 Agent 列表
```
